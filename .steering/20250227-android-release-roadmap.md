# Android向けGoogle Playストアリリース ロードマップ

**作成日**: 2025-02-27
**目的**: Godot 4.3で開発中の2Dローグライトゲームを、ユーザー満足度の高い品質でGoogle Playストアにリリースする

---

## 現状分析（2025-02-27時点）

### ✅ 実装済み
- コアゲームループ（移動、自動攻撃、経験値、レベルアップ）
- 武器システム（7種類: Straight Shot, Area Blast, Homing Missile, Orbital, Laser Beam, Lightning, 高速連射）
- 敵システム（Basic, Fast, Heavy, Strong + ボス3種: Tank, Sniper, Swarm）
- アップグレードシステム（3択選択）
- スキルシステム（4種: Dash, Nova Blast, Shield, Time Slow + クールダウンUI）
- プールマネージャー（敵・弾丸の最適化）
- ハザードシステム（Lava Pool, Ice Patch, Poison Cloud）
- パワーアップシステム（Speed Boost, Invincibility, Double Damage, Magnet）
- コンボシステム（連続撃破で倍率アップ）
- UI（HUD, ゲームオーバー画面、タイトル画面）

### ⚠️ 未実装・課題
- **モバイル操作**: タッチ入力対応なし（キーボードのみ）
- **サウンド**: プレースホルダー音のみ
- **ビジュアル**: プレースホルダーグラフィック（ColorRect）
- **パフォーマンス**: モバイル端末での動作未検証
- **Google Play対応**: ストア掲載素材なし
- **多言語対応**: 日本語のみ
- **チュートリアル**: なし
- **セーブデータ**: なし（リトライのみ）
- **法務対応**: プライバシーポリシー、利用規約なし

## **Phase 1: モバイル基盤整備（2-3週間）**

### 目的
Android端末で快適にプレイできる基盤を構築

### 1.1 タッチ入力対応（優先度: 最高）

**タスク**
- [ ] バーチャルジョイスティック実装
  - 画面左側: 移動用スティック
  - 画面右側: スキル発動ボタン
- [ ] タッチ感度調整（デッドゾーン、最大範囲）
- [ ] 入力抽象化レイヤー（キーボード・タッチ両対応）
- [ ] タブレット対応（画面サイズ別レイアウト）

**実装方針**
```gdscript
# autoload/input_manager.gd
extends Node

var virtual_joystick_active: bool = false
var movement_vector: Vector2 = Vector2.ZERO

func get_movement_input() -> Vector2:
    if OS.has_feature("mobile"):
        return movement_vector  # タッチ入力
    else:
        return Input.get_vector("ui_left", "ui_right", "ui_up", "ui_down")
```

**検証項目**
- [ ] 5インチスマホで快適に操作可能
- [ ] 10インチタブレットでレイアウト崩れなし
- [ ] 60FPS維持（Pixel 6a相当の端末）

---

### 1.2 パフォーマンス最適化（優先度: 最高）

**ターゲット端末**
- **最低**: Snapdragon 665相当（Redmi Note 8など）
- **推奨**: Snapdragon 778G相当（Pixel 6a、Galaxy A53など）

**タスク**
- [ ] プロファイリング実行（Godot Profiler + Android adb）
- [ ] 描画最適化
  - [ ] スプライトバッチング有効化
  - [ ] 画面外オブジェクトの描画停止（VisibleOnScreenNotifier2D）
  - [ ] パーティクル数削減（モバイル設定）
- [ ] 物理演算最適化
  - [ ] コリジョンレイヤー最適化
  - [ ] 物理演算フレームレート調整（60Hz → 30Hz検討）
- [ ] メモリ最適化
  - [ ] テクスチャ圧縮（ETC2）
  - [ ] プール上限調整（敵200→100、弾500→300）
  - [ ] GC頻度削減（オブジェクト生成削減）

**パフォーマンス目標**
| 端末クラス | フレームレート | メモリ使用量 | バッテリー消費 |
|-----------|---------------|-------------|---------------|
| 最低スペック | 30 FPS安定 | < 300MB | < 10%/時間 |
| 推奨スペック | 60 FPS安定 | < 512MB | < 8%/時間 |

### 4.3 APK/AAB ビルド設定（優先度: 最高）

**タスク**
- [ ] Godot Export Template ダウンロード（Android）
- [ ] keystore生成（署名用）
  ```bash
  keytool -genkey -v -keystore release.keystore -alias mygame \
    -keyalg RSA -keysize 2048 -validity 10000
  ```
- [ ] build.gradle 設定
  - minSdkVersion: 21（Android 5.0）
  - targetSdkVersion: 34（Android 14、最新）
  - versionCode: 1
  - versionName: "1.0.0"
- [ ] 権限設定（最小限）
  ```xml

  ```
- [ ] Android App Bundle (AAB) 生成
  ```bash
  godot --headless --export-release "Android" ./build/game.aab
  ```

- インストールサイズ: < 100MB

### 高リスク項目
1. **パフォーマンス不足**
   - 対策: 早期に実機テスト、プール上限削減
   - 代替案: グラフィック品質低下、エフェクト削減

2. **審査リジェクト**
   - 対策: 事前にポリシー熟読、内部テスト徹底
   - 代替案: リジェクト理由に応じて迅速修正

3. **アセット制作遅延**
   - 対策: 有料素材パック購入で時間短縮
   - 代替案: MVP版でプレースホルダー継続使用

### 中リスク項目
4. **ローカライズ品質**
   - 対策: DeepL + ネイティブレビュー
   - 代替案: 日本語のみリリース

5. **バグ多発**
   - 対策: QA期間2週間確保
   - 代替案: リリース後にホットフィックス
